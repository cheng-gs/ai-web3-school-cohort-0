# AI 基础概念整理

这份笔记用于建立后续学习 learning agent、workflow、AI coding 和工具调用时的共同语言。内容以自己的理解为主

## 1. LLM

### 一句话解释

LLM 是一种根据上下文预测下一个 token 的模型，它擅长生成语言、解释问题和组织信息，但不是“知道一切”的事实数据库。

### 具体例子

我把一段智能合约代码贴给 LLM，让它解释每个函数在做什么，它通常能快速给出结构化说明，帮助我先建立整体理解。

### 常见误区或边界

常见误区是把 LLM 当成绝对可靠的知识源。它可能会一本正经地说错，所以涉及事实、链接、链上地址、API 参数时必须再验证。

## 2. Prompt

### 一句话解释

Prompt 是我给模型的任务说明，它决定模型当前该做什么、按什么方式输出，而不是永久改变模型本身。

### 具体例子

如果我只说“解释钱包”，模型可能会回答得很散；如果我说“用给 AI 背景、Web3 新手的角度，用 3 句话解释钱包，并补一个 Web2 对比”，输出通常会更稳定。

### 常见误区或边界

误区是以为 prompt 越长越好。实际上，不清楚的长 prompt 往往比简洁明确的 prompt 更差，重点是任务、对象、输出格式和边界要清楚。

## 3. Context Window

### 一句话解释

Context window 可以理解为模型当前这一轮工作时能“看到”的输入范围，它决定模型这次回答能参考多少信息。

### 具体例子

我把一整周的课程内容、笔记和代码都塞给模型时，如果超出上下文能力，模型可能只抓住后半段内容，前面的要求会被忽略或遗忘。

### 常见误区或边界

误区是以为“模型看过一次就会永久记住”。上下文窗口只对当前会话或当前轮次有效，超出范围的信息可能被截断，跨会话通常也不会自动保留。

## 4. Workflow

### 一句话解释

Workflow 是预先设计好的任务流程，步骤相对固定，模型只是其中一个执行节点，而不是自己决定全部路径。

### 具体例子

例如我的学习 workflow 可以是：读取课程要求 -> 生成今日任务 -> 我人工确认 -> 写入 daily note -> 检查 git status -> 我确认后再 commit。这里每一步顺序都比较明确。

### 常见误区或边界

误区是把任何多步骤任务都叫 agent。其实很多场景只是 workflow，因为路径和顺序早就写死了，不需要模型自主规划。

## 5. Agent

### 一句话解释

Agent 是能根据目标自主拆解任务、决定下一步并调用工具完成工作的模型系统，但它的自主性越高，越需要边界和监控。

### 具体例子

我让 learning agent 帮我初始化学习仓库、生成计划、维护 daily note、整理 Handbook feedback，这比单次问答更像 agent，因为它要持续处理任务并结合上下文做决策。

### 常见误区或边界

误区是觉得 agent 一定比 workflow 高级、一定更好。实际上，如果任务路径很清楚，用固定 workflow 往往更稳定、更容易调试，也更安全。

## 6. Tool Use

### 一句话解释

Tool use 是模型不只“说”，而是通过结构化方式调用外部工具来“做事”，例如读文件、查网页、运行命令或写入系统。

### 具体例子

在这个仓库里，agent 通过工具读取目录、创建 Markdown 文件、执行 `git status`，这就是典型的工具调用，而不是纯文本聊天。

### 常见误区或边界

误区是只要接上工具，模型就会自动可靠。实际上，工具调用只是让模型获得执行能力，调用错工具、传错参数、越权操作的风险反而更高。

## 7. AI Coding

### 一句话解释

AI coding 是用模型辅助写代码、读代码、改代码、补文档或生成脚手架，以提升开发效率，但不是把工程判断完全外包给模型。

### 具体例子

我可以让 AI 帮我先生成一个最小合约交互脚本，或者先整理一个 repo 结构草稿，再由我自己检查命名、依赖、风险和可维护性。

### 常见误区或边界

误区是“能跑起来就说明是对的”。AI 生成的代码可能只是恰好运行，不代表安全、鲁棒、可维护，也不代表适合真实链上操作。

## 8. Guardrails

### 一句话解释

Guardrails 是给模型或 agent 设定的输入输出和执行边界，用来防止它做不该做的事。

### 具体例子

在 AI × Web3 场景里，最重要的 guardrail 之一就是：涉及签名、授权、转账、部署和合约写入时，必须停下来由人确认，不能让 agent 自动越过这一步。

### 常见误区或边界

误区是把 guardrails 当成一句“请注意安全”的提示。真正有效的 guardrails 应该能阻止动作，而不是只做口头提醒。

## 9. Tracing

### 一句话解释

Tracing 是把 agent 的执行过程记录下来，让我能回头看它做了哪些步骤、调用了哪些工具、为什么得到这个结果。

### 具体例子

如果一个 agent 生成了错误的测试网交互脚本，我需要知道它是读错了文档、理解错了 ABI，还是工具参数传错了，这就需要 tracing。

### 常见误区或边界

误区是只看最终输出，不看过程。越是多步骤任务，越要能看过程，否则一旦出错，很难定位到底是哪一层出了问题。

## 10. Human-in-the-loop

### 一句话解释

Human-in-the-loop 是把人放回关键决策节点，让模型负责提议或准备材料，但关键动作必须由人审核和确认。

### 具体例子

例如 AI 可以先生成合约部署步骤和交易说明，但真正点击钱包确认、发送交易、记录验证结果这一步，必须由我自己完成。

### 常见误区或边界

误区是把人工确认理解成“最后随便看一眼”。真正的人在环应该承担判断责任，尤其在资金、安全、授权、公开发布等高风险节点上不能走过场。
